# FAST REORIENTATION

Use this file when starting a new chat, migrating from a saturated chat, changing machines, or recovering project continuity.

## REORIENTATION ORDER

1. Read PROJECT_INSTRUCTIONS.txt if available.
2. Read FOUNDATION_CONTEXT.md.
3. Read PROJECT_OVERLAY.md.
4. Read PROJECT_STATUS.md.
5. Identify the next concrete action.
6. Continue in small validated units.

## DO NOT

- Do not assume full chat history is available.
- Do not import rules from other projects.
- Do not overwrite project overlays without explicit instruction.
- Do not replace operational data unless the user confirms.
- Do not continue redesigning FOUNDATION unless the user specifically asks.

## CONTINUITY RULE

If the user says "continue", proceed with the next concrete action described in PROJECT_STATUS.md.

If PROJECT_STATUS.md is missing or outdated, ask for or reconstruct the current state before making changes.

## MIGRATION RULE

When saturation risk is high, recommend migration and prepare a concise status update in PROJECT_STATUS.md.
